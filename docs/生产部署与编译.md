# 生产部署与编译

## 环境要求

- Go 1.25 或更高版本。
- Node.js 最新稳定版。
- npm。
- Windows 管理员权限或 Linux/macOS root 权限，用于绑定 DHCP/TFTP 等低端口。

## 本地生产构建

Windows：

```powershell
cd pxe
npm ci --prefix web
npm run build --prefix web
go test ./...
go vet ./...
New-Item -ItemType Directory -Force -Path dist | Out-Null
go build -trimpath -ldflags="-s -w" -o dist\pxe.exe .\cmd\pxe
```

Linux/macOS：

```bash
cd pxe
(cd web && npm ci && npm run build)
go test ./...
go vet ./...
mkdir -p dist
go build -trimpath -ldflags="-s -w" -o dist/pxe ./cmd/pxe
```

`-trimpath -ldflags="-s -w"` 是生产构建默认参数，用于去除本地路径、符号表和调试信息，减小二进制体积，降低首次启动时被系统安全扫描拖慢的概率。

## 交叉编译

前端构建会直接输出到 `internal/web/dist`，之后可交叉编译：

```powershell
$env:CGO_ENABLED="0"
$env:GOOS="linux"
$env:GOARCH="arm64"
New-Item -ItemType Directory -Force -Path dist | Out-Null
go build -trimpath -ldflags="-s -w" -o dist\pxe-linux-arm64 .\cmd\pxe
Remove-Item Env:\GOOS
Remove-Item Env:\GOARCH
Remove-Item Env:\CGO_ENABLED
```

纯 Go SQLite 使用 `modernc.org/sqlite`，不依赖 CGO，适合 Windows、Linux、macOS、OpenWrt、Armbian 等环境交叉编译。

## 生产运行建议

- ProxyDHCP 模式适合已有路由器 DHCP 的网络，是默认推荐模式。
- 完整 DHCP 模式只建议在隔离网络中使用，避免与路由器 DHCP 冲突。
- 管理端默认监听 `127.0.0.1:8088`，远程管理必须启用登录认证并限制访问来源。
- Windows 需要允许 UDP 67、UDP 69、UDP 4011 和 HTTP Boot 端口通过防火墙。
- Linux 建议使用 root 运行，或为二进制授予绑定低端口所需 capability。
- 大镜像文件放在文件系统中，不放数据库。

## GitHub 发布流水线

`.github/workflows/release.yml` 是手动触发流水线：

1. 输入版本号，例如 `v1.0.0`。
2. 执行前端类型检查、前端构建、Go 测试、`go vet`。
3. 构建 Windows、Linux、macOS 多平台二进制。
4. 打包为 zip/tar.gz。
5. 创建 GitHub Release 并上传产物。

Dependabot 每月检查 Go、npm 和 GitHub Actions 依赖更新。
