# 离线自托管启动文件和镜像

`pxe` 支持完全离线使用。核心原则是：启动加载器、iPXE 脚本、ISO/WIM/IMG/VHD 等镜像都放在本机文件系统中，由内置 TFTP 和 HTTP Boot 服务提供。

## 目录约定

```text
data/boot/
├─ netboot/
│  ├─ netboot.xyz.kpxe
│  ├─ netboot.xyz-undionly.kpxe
│  └─ netboot.xyz.efi
├─ tftp/
│  └─ 自定义 TFTP 启动文件
└─ http/
   ├─ images/
   │  ├─ winpe.iso
   │  ├─ rescue.wim
   │  └─ linux.iso
   └─ Boot/
      └─ ipxefm/
```

`boot/netboot` 是 netboot.xyz 下载目录。程序会把它虚拟暴露为：

```text
tftp://服务IP/netboot/netboot.xyz.kpxe
http://服务IP/netboot/netboot.xyz.kpxe
```

因此不需要把 netboot 文件复制到 `boot/tftp` 或 `boot/http`。

## 在线准备，离线使用

1. 在有互联网的机器上打开 Web UI。
2. 进入 `netboot.xyz` 页面。
3. 下载 `netboot.xyz.kpxe`、`netboot.xyz-undionly.kpxe`、`netboot.xyz.efi`。
4. 把整个 `data/boot/netboot` 目录复制到离线服务器相同位置。
5. 把需要启动的 ISO/WIM/IMG/VHD 放入 `data/boot/http`。
6. 在启动菜单中添加对应文件路径。

## 完全手动准备

也可以手动下载文件并放入：

```text
data/boot/netboot/
```

默认来源为：

```text
https://boot.netboot.xyz/ipxe
```

如果外网地址变化，可以在 Web UI 的 netboot.xyz 设置中调整 Base URL。

## 离线镜像启动建议

- 小型 NBP、EFI、kpxe 文件适合通过 TFTP 提供。
- ISO、WIM、VHD、IMG 等大文件应通过 HTTP Boot 提供。
- HTTP Boot 支持 Range，适合大文件和部分挂载场景。
- 离线环境中，菜单项应使用相对路径或本机 HTTP 地址，避免引用公网 URL。

示例菜单路径：

```text
images/winpe.iso
images/rescue.wim
images/linux.iso
```

## 排障

- DHCP 日志能看到客户端请求，但没有 TFTP 记录：检查 ProxyDHCP 响应、VirtualBox/交换机/防火墙是否允许广播。
- TFTP 下载完成但不继续：确认 DHCP 下发的是可执行 NBP，不是普通脚本。
- HTTP 下载失败：确认通告 IP 是客户端能访问到的服务器网卡 IP。
- netboot.xyz 进入后仍访问外网：这是 netboot.xyz 菜单自身行为；完全离线时应使用本地镜像菜单。
