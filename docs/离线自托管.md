# 离线自托管安装

`pxe` 可以在完全离线环境安装系统。推荐方式是：老式 BIOS 先加载干净的 iPXE，再从本机 HTTP 目录加载 `boot.ipxe`、内核和 initrd。

其中 `192.168.137.1` 应替换为 Web UI 中配置的通告 IP。

## BIOS 离线安装步骤

1. 下载官方 BIOS iPXE 文件：https://boot.ipxe.org/undionly.kpxe 并改名为 `netboot.xyz-undionly.kpxe`，然后放到 `data/boot/netboot/` 目录里。BIOS 启动文件优先级是 `netboot.xyz.kpxe`、`netboot.xyz-undionly.kpxe`、`ipxe.bios`；如果要强制使用 `netboot.xyz-undionly.kpxe`，不要同时保留更高优先级的 `netboot.xyz.kpxe`。

2. 然后把系统安装文件放到 http 目录，并新增一个 `boot.ipxe` 文件：

```text
data/boot/http/boot.ipxe
data/boot/http/linux
data/boot/http/initrd.gz
```

3. `boot.ipxe` 内容示例：

```ipxe
#!ipxe
set base-url http://192.168.137.1
kernel ${base-url}/linux initrd=initrd.gz ip=dhcp
initrd ${base-url}/initrd.gz
boot
```

4. 启动菜单里，iPXE 菜单项默认使用 `Run boot.ipxe`，启动文件或脚本为：

```text
%dynamicboot%=boot.ipxe
```

5. 保存菜单，启动 PXE 客户端，进入本项目 iPXE 菜单后选择该菜单项。

## 重要注意

- 离线安装更推荐使用发行版提供的 `linux` + `initrd.gz`，并在 `boot.ipxe` 中明确写启动参数。
- DHCP 通告 IP 必须是客户端能访问到的服务器网卡 IP，否则会出现 TFTP 或 HTTP 超时。
- Windows 防火墙需要放行 UDP 67、UDP 69、UDP 4011 和 TCP 80。

## netboot.xyz 本地钩子

点击 Web UI 的 `netboot.xyz -> 下载文件` 后，程序会自动检查：

```text
data/boot/tftp/local-vars.ipxe
```

如果文件不存在，会自动生成；如果已经存在，不会覆盖，所以你也可以自己写这个引导脚本。netboot.xyz 在本地 DHCP/ProxyDHCP 引导场景中会尝试读取这个文件，因此可以用它进入本项目提供的英文安装菜单。

默认菜单包含两个镜像：

```text
Public Install Debian 12
Public Install Alpine Linux
Local Install Debian 12
Local Install Alpine Linux
iPXE Shell
Continue netboot.xyz
```

公网安装会直接从清华镜像站下载内核和 initrd。内网安装会从通告 IP 和当前 HTTP Boot 端口下载同样路径的文件；如果 HTTP Boot 端口不是 80，会自动追加端口，例如：

```text
http://通告IP/debian/dists/bookworm/main/installer-amd64/current/images/netboot/debian-installer/amd64/linux
http://通告IP/debian/dists/bookworm/main/installer-amd64/current/images/netboot/debian-installer/amd64/initrd.gz
http://通告IP/alpine/v3.23/releases/x86_64/netboot/vmlinuz-lts
http://通告IP/alpine/v3.23/releases/x86_64/netboot/initramfs-lts
http://通告IP:8080/debian/dists/bookworm/main/installer-amd64/current/images/netboot/debian-installer/amd64/linux
```

对应文件需要放在 `data/boot/http` 下的相同相对路径中。

## 排障

- HTTP 文件打不开：先用浏览器访问 `http://通告IP`查看是否有文件。
- 内核启动失败：检查 `boot.ipxe` 中的内核参数是否符合当前发行版要求。
